## 1. 文档信息

| 项目       | 内容               |
| ---------- | ------------------ |
| 产品名称   | Panda Island       |
| 内部项目名 | `panda-desktop`    |
| 文档版本   | 0.1.0              |
| 产品阶段   | Windows MVP        |
| 目标系统   | Windows 10/11 x64  |
| 文档状态   | 核心产品决策已确认 |

## 2. 产品定义

Panda Island 是一款常驻 Windows 桌面的轻量 2D 熊猫应用。它不追求复杂游戏系统，而是通过自然的桌面陪伴、可控的互动和可信的熊猫资料，形成“桌宠 + 科普入口”的基础体验。

首版验证三个问题：

1. 透明桌宠是否能稳定存在于 Windows 日常桌面环境中。
2. 简单动画和点击互动是否足以形成陪伴感。
3. 资料卡和公开直播外链是否能为后续熊猫内容产品提供有效入口。

## 3. 产品目标

### 3.1 MVP 目标

- 用户安装后能在 1 分钟内看到并使用熊猫。
- 熊猫窗口不破坏桌面视觉，能拖动并保持在用户选择的位置。
- 熊猫具备待机、走路、睡觉、吃竹子和点击反馈五类可识别表现。
- 用户能查看一只模拟熊猫的完整资料。
- 用户能明确看到直播来源并主动跳转到外部浏览器。
- 用户设置在重启应用后仍然有效。
- 首版只提供一只暂名“花花”的原创卡通数字分身，默认开启低音量互动声音。

### 3.2 非目标

第一版不实现：

- 实时视频识别或直播行为识别；
- 抓取、转播或嵌入第三方直播流；
- 全球在世大熊猫数据库；
- AI 对话、AI 实时动画或 AI 健康判断；
- 用户登录、云同步、好友、社交和排行榜；
- 货币、付费、抽卡、任务或复杂养成数值；
- 3D 熊猫岛、多人联机；
- Windows ARM64、Windows 7/8、macOS 或 Linux。

## 4. 目标用户与使用场景

### 4.1 核心用户

- 喜爱大熊猫、希望获得轻量桌面陪伴的用户；
- 长时间使用电脑，希望有低打扰互动的学生和办公人群；
- 对熊猫资料和公开直播入口感兴趣的内容用户。

### 4.2 核心场景

| 场景     | 用户行为                         | 产品响应                                       |
| -------- | -------------------------------- | ---------------------------------------------- |
| 首次启动 | 用户运行应用                     | 在主屏幕右下安全区域显示熊猫并给出简短引导     |
| 日常陪伴 | 用户继续处理其他工作             | 熊猫在低频状态机驱动下待机、走动、吃竹子或睡觉 |
| 点击互动 | 用户单击熊猫                     | 熊猫进入 `react` 状态并播放反馈动画            |
| 移动位置 | 用户按住熊猫拖动                 | 窗口跟随鼠标，释放后保存位置                   |
| 查看资料 | 用户双击熊猫或选择右键菜单“资料” | 打开资料卡窗口，不关闭桌宠                     |
| 查看直播 | 用户点击“观看公开直播”           | 先显示来源，再经确认在默认浏览器打开白名单链接 |
| 临时安静 | 用户在托盘或右键菜单暂停动画     | 熊猫保持待机，不再自动切换状态                 |
| 退出     | 用户选择“退出”                   | 关闭所有窗口、托盘和后台进程                   |

| PET-005 | 用户可按住熊猫可交互区域拖动窗口                  | P0          |
| PET-006 | 释放拖动后保存窗口位置                            | P0          |
| PET-007 | 单击熊猫进入点击反馈状态                          | P0          |
| PET-008 | 双击熊猫打开资料卡                                | P0          |

## 6. 用户流程

### 6.1 首次启动

1. 应用检查本地设置文件。
2. 若无设置，创建默认设置。
3. 读取本地熊猫数据和默认熊猫资源。
4. 在主显示器工作区右下方显示桌宠，避开任务栏。
5. 显示一次性轻提示：“拖动我，双击查看资料，右键打开菜单”。
6. 初始化托盘。

### 6.2 外链直播

1. 用户在资料卡点击“观看公开直播”。
2. 弹窗显示提供方名称和即将访问的域名。
3. 用户确认。
4. 应用验证 URL 为 HTTPS 且符合配置白名单。
5. 使用系统默认浏览器打开。
6. Panda Island 不读取、不代理、不记录直播内容。

## 7. 体验原则

- **低打扰**：状态切换频率低，不抢焦点，不主动播放高音量声音。
- **可预期**：用户拖动、关闭资料卡和退出的结果必须明确。
- **可信**：资料必须显示来源、审核状态和更新时间。
- **克制**：首版不加入伪养成数值、诱导签到和强通知。
- **可替换**：角色资产与行为逻辑分离，新增熊猫不需要修改状态机。

## 8. 非功能需求

| 类别     | 目标                                                             |
| -------- | ---------------------------------------------------------------- |
| 启动性能 | 常规 SSD 设备冷启动到显示熊猫不超过 3 秒                         |
| 空闲资源 | 单只熊猫静态/待机时 CPU 平均占用目标低于 2%，内存目标低于 150 MB |
| 动画性能 | 60 Hz 显示器上目标 30 FPS 稳定播放，无持续掉帧                   |
| 稳定性   | 连续运行 8 小时无崩溃、无明显内存持续增长                        |
| 离线能力 | 除主动打开外链外，核心功能完全离线可用                           |
| 安全     | 前端不获得任意 shell、文件系统或任意 URL 打开权限                |
| 可访问性 | 资料卡支持键盘操作、清晰焦点和 125%–200% Windows 缩放            |
| 隐私     | MVP 不收集账号、设备标识、行为日志和个人数据                     |

## 9. 内容、版权与合规边界

- MVP 角色暂名“花花”，可参考真实大熊猫花花圆脸、短腿、体态圆润等公开特征，但必须重新设计成具有明显原创性的卡通数字分身。
- 不直接描摹真实照片，不复制动物园、媒体或第三方已有卡通形象，不宣称为官方授权数字形象。
- 每条熊猫资料保留来源、访问时间、审核状态、公开和商业使用标记。
- 不把第三方直播嵌入客户端；只在用户确认后跳转合法公开页面。
- 正式上线前须完成名称、图像、字体、音效、直播入口和数据来源授权清单。

## 10. MVP 验收标准

版本只有在以下条件全部满足后才能标记为 MVP：

1. Windows 10 与 Windows 11 实机均可安装、启动和退出。
2. 熊猫窗口透明、无边框、置顶并可拖动。
3. 六种状态符合状态机规则，测试使用固定随机源时结果可重复。
4. 点击、双击和拖动之间无明显冲突。
5. 资料卡信息完全来自本地 JSON，错误数据会被拒绝加载。
6. 外链经过来源提示、用户确认和白名单校验。
7. 设置、熊猫选择和窗口位置在重启后恢复。
8. 资料卡关闭不会退出主应用；退出会终止全部进程。
9. 类型检查、单元测试、Rust 测试和 Windows 构建全部通过。
10. 完成一次 8 小时稳定性观察并记录资源占用。

## 11. 已确认产品决策

1. 首版只提供 1 只熊猫。
2. 角色暂名“花花”，以真实大熊猫花花的公开体态特征为灵感，制作原创卡通数字分身。
3. 声音默认开启，默认音量 35，用户可静音或调节。
4. 首版只提供简体中文。
5. 首轮交付 Windows x64 NSIS 测试安装包，不进入 Microsoft Store。
