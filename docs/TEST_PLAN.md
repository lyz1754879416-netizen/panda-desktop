# Panda Island Windows MVP 测试与验收计划

## 1. 测试目标

测试不仅验证 React 页面，还要验证 Windows 原生窗口、托盘、进程生命周期、DPI、多显示器和安装包。透明、置顶和点击穿透必须以实际运行结果为准。

## 2. 测试分层

| 层级                | 工具/方式             | 重点                                                 |
| ------------------- | --------------------- | ---------------------------------------------------- |
| TypeScript 静态检查 | `tsc --noEmit`        | 严格类型、无隐式 any、接口一致性                     |
| 单元测试            | Vitest                | 状态机、随机源、时长、数据校验、设置迁移、URL 白名单 |
| 组件测试            | React Testing Library | 资料卡、设置、来源确认、键盘操作、错误状态           |
| Tauri API 适配测试  | Vitest + Tauri mock   | 窗口、Store、Opener、Autostart 调用协议              |
| Rust 测试           | `cargo test`          | URL 校验、位置校正、窗口/设置命令中的纯逻辑          |
| 前端构建            | Vite build            | 生产资源和路由                                       |
| 原生构建            | Tauri build/check     | Rust、配置、capabilities、Windows bundle             |
| Windows 集成测试    | 自动化 + 手工         | 透明窗口、鼠标、托盘、多窗口和生命周期               |
| 稳定性测试          | 资源监控 + 长时间运行 | 内存、CPU、计时器、纹理和句柄泄漏                    |

## 3. 单元测试设计

### 3.1 行为状态机

必须覆盖：

- 初始状态恒为 `idle`；
- `idle` 时长最小 8 秒、最大 20 秒；
- 固定随机序列能得到固定下一状态和时长；
- `sleep` 范围 20–60 秒，`eat` 范围 8–15 秒；
- 点击可打断 `idle/walk/sleep/eat` 并进入 `react`；
- 拖动可从任何状态进入 `dragged`；
- 释放拖动回到 `idle`；
- `react` 动画完成回到 `idle`；
- 暂停取消计时器，恢复后只有一个有效计时器；
- 状态切换后旧 generation token 事件无效；
- 销毁调度器后没有挂起计时器；
- 不连续三次选择同一自动状态。

### 3.2 数据与设置

- 合法 `PandaProfile` 解析成功；
- 缺失必填字段、错误日期、非法 ID、HTTP 链接被拒绝；
- 未审核资料不进入公开列表；
- `embedAllowed: true` 在 MVP schema 中被拒绝；
- 设置音量自动限制到 0–100；
- 未知熊猫 ID 回退到默认熊猫；
- V1 设置可序列化并往返一致；
- 损坏 JSON 恢复默认设置；
- 未来 schema 版本不会被静默错误解析。

### 3.3 外链安全

- 允许白名单域名下的 HTTPS URL；
- 拒绝 HTTP、`file:`、`javascript:`、`data:` 和自定义 scheme；
- 拒绝相似域名，例如 `allowed.example.evil.com`；
- 拒绝 URL 用户信息混淆和非标准端口（除非明确允许）；
- 必须经过用户确认后才调用 Opener；
- 来源禁用或过期时按钮不可用。

## 6. 安装、升级与卸载

- 干净 Windows 用户账户安装 NSIS 包；
- 普通用户权限安装和启动，不请求管理员权限（除非安装方式明确需要）；
- WebView2 缺失时给出可理解处理；
- 从上一测试版本覆盖升级，设置不丢失；
- 卸载后程序文件和开机启动项被移除；
- 是否保留用户设置由卸载界面或文档明确说明；
- 未签名内测包记录 SmartScreen 现象，公开版验证数字签名。

## 7. 性能与稳定性

### 7.1 性能基线

- 冷启动到熊猫可见：目标 ≤ 3 秒；
- 待机 CPU：平均目标 < 2%；
- 内存：目标 < 150 MB；
- 动画：目标 30 FPS，正常桌面负载下无连续明显掉帧；
- 隐藏后：调度器暂停，CPU 接近空闲；
- 资料卡首次打开：目标 ≤ 1 秒。

### 7.2 8 小时稳定性

测试期间每 30 分钟记录：

- 工作集内存；
- CPU 平均/峰值；
- GPU 内存（可获取时）；
- 线程、句柄数量；
- 当前状态和状态切换次数；
- 是否发生黑底、闪烁、动画停滞或窗口丢失。

判定：无崩溃；内存、线程和句柄不存在持续单调增长；应用仍能点击、拖动、打开资料卡和退出。

## 8. 安全检查

- 审查所有 Tauri capabilities，只授予目标窗口必要权限；
- 确认未启用通用 shell 执行；
- CSP 不允许远程脚本和任意连接；
- 外链域名列表与熊猫数据来源清单一致；
- 数据和资源路径不能目录穿越；
- 日志不输出用户隐私和完整本机路径；
- 依赖审计无已知高危漏洞，无法立即修复的风险要记录并获准。

## 11. Stage 0 文档一致性验收

- PRD 中的 P0 功能在 ROADMAP 中都有实施阶段；
- PRD 验收项在本测试计划中都有验证方法；
- ARCHITECTURE 的模块与目录能承载 DATA_MODEL；
- 状态名称和时长在全部文档中一致；
- Windows-only 范围在所有文档中一致；
- 点击穿透明确为 P1，不与 P0 拖动和点击冲突；
- 外链、资料来源和授权边界在产品、架构、数据和测试中一致。
- 首版角色固定为 `panda_huahua_mvp`，界面为简体中文，默认声音开启，构建目标为 NSIS x64 测试安装包。
